# ADR-013: Product Positioning — Workflow Control Plane

状态: 已确认
日期: 2026-04

## 背景

Sopify 需要明确自身在 AI 编程生态中的定位，避免与宿主（Claude Code、Codex、Cursor）、技能市场（Superpowers）或方法论产品（Spec-Kit）竞争。

## 决策

**Sopify 是 AI 编程工作流的 control plane。**

Core 职责：

1. **自适应工作流**：按任务复杂度选择快速修复 / 轻量迭代 / 完整方案
2. **状态交接**：handoff 机器契约让任务跨轮、跨会话可恢复
3. **质量治理**：独立验证闭环（cross-review 是参考实现，不是 core 功能）
4. **资产沉淀**：blueprint / history 构建跨任务项目记忆
5. **宿主无关**：`.sopify-skills/` 纯文件协议

不做的事：

| 领域 | 不做 | 归属 |
|------|------|------|
| 模型推理 | LLM 调用、代码生成 | 宿主 |
| 技能市场 | skill 分发、安装、计费 | 独立生态 |
| 方法论 | spec 写作、设计模式教学 | 独立产品 |
| 代码执行 | 文件编辑、终端操作 | 宿主 |

## 生存性测试

2027 年宿主原生支持 plan/checkpoint/multi-agent 后，Sopify 仍必须保留：

- 项目级资产沉淀（blueprint / history）
- 跨宿主连续工作
- 可审计决策链
- 独立质量闭环
- 计划漂移治理

如果以上任一能力被宿主完全替代且无跨宿主可携带性需求，该能力应 sunset。

## 后果

- Core 只保留协议/状态/权限层
- 具有独立用户价值的分析/推理能力设计为可外部化组件
- CrossReview 是已实现的可外部化参考范本
