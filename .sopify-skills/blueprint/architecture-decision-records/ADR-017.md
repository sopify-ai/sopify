# ADR-017: Action/Effect Boundary

状态: P0 完成，持续扩展
日期: 2026-04-28 (P0)

## 背景

在 ActionProposal 之前，Host LLM 直接驱动 runtime 路由和副作用。局部语境请求被误读为全局推进是通用问题。需要在 materialization 之前建立授权边界。

## 决策

**在 State Resolution 完成后、Router 产生副作用之前，插入 ActionProposal 解析与 Effect 授权。**

核心管线：

```
用户自然语言
  → Host LLM 映射为 ActionProposal（action_type + side_effect + confidence + evidence）
  → Validator 基于 ActionProposal + ValidationContext 输出 ValidationDecision
  → Deterministic action 执行
  → Handoff / Receipt 暴露机器事实
```

### 不变量

1. **Host LLM 只是 proposal source，不是 authorizer** — LLM 提交工单，不批准工单
2. **Validator 是唯一授权者** — 判断当前 context 下 action/side effect 是否允许
3. **Validator 不是 executor** — 不做 plan materialization、文件迁移、自动修复、状态推进
4. **执行层不理解人话** — 只按结构化字段和文件事实做事
5. **`fallback_router` 是临时兼容出口** — 应单调收缩，不承接新的长期能力

### ExecutionAuthorizationReceipt（方向）

execution_confirm checkpoint 重分类为机器授权事实：

**不变量：**
- 绑定 plan identity + plan revision + execution gate result + action proposal identity + authorization source
- 使用 canonical JSON + sha256 生成 fingerprint
- Plan 变更后 receipt 自动失效
- Fail-closed：任一字段不匹配则拒绝执行

**Receipt 字段定义：**

| 字段 | 语义 |
|------|------|
| `plan_id` | 目标 plan 的唯一标识 |
| `plan_path` | 目标 plan 的文件路径 |
| `plan_revision_digest` | plan 内容的 sha256 摘要 |
| `gate_status` | execution gate 的判定结果 |
| `action_proposal_id` | 触发授权的 ActionProposal 标识 |
| `authorization_source` | `{ kind: "host_turn", turn_id }` 或 `{ kind: "request_hash", request_sha1 }` |
| `fingerprint` | `sha256(canonical_json({plan_id, plan_path, plan_revision_digest, gate_status, action_proposal_id}))` |
| `authorized_at` | ISO 8601 时间戳 |

Implementation plan 可补充字段，但不得删除或弱化上述字段的 fail-closed 语义。

### Checkpoint 重分类

| 旧类型 | 新定位 | 理由 |
|--------|--------|------|
| `clarification` | **保留为 canonical checkpoint** | 补事实是真协作分叉 |
| `decision` | **保留为 canonical checkpoint** | 拍板选路是真协作分叉 |
| `plan_proposal` | propose_plan pending artifact | ~~等 side-effect proof 稳定后降级~~ **Wave 3a 已 hard-cut 删除** |
| `execution_confirm` | ExecutionAuthorizationReceipt | 授权不是协作分叉 |
| `develop_checkpoint` | develop callback source | 触发 clarification/decision，不是独立类型 |

## 后续扩展方向

- `propose_plan` + `write_plan_package` side-effect proof
- `execute_existing_plan` 通过 ExecutionAuthorizationReceipt 授权
- `fallback_router` 职责单调收缩

## 后果

- 所有 side-effecting action 必须先映射为 ActionProposal
- `~go finalize` 等 command alias 必须映射为对应 proposal，不绕过 Validator
- Checkpoint 目标压缩为 2 种 canonical type
- required_host_action 目标压缩为 5 种 canonical action
